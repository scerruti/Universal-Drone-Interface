# Contributing to Universal-Drone-Interface

Thank you for your interest in contributing! Please review the following guidelines to help us maintain a high-quality, collaborative project.

## How to Contribute
- **Bug Reports & Feature Requests:** Open a GitHub issue and use the provided templates.
- **Pull Requests:** Fork the repo, create a feature branch, and submit a pull request with a clear description.
- **Discussions:** Use issues for questions, suggestions, or API design discussions.

## Project Structure
- `STANDARD_DRONE_API_PROPOSAL.md`: API design and interface proposal
- `DRONE_*_COMPARISON.md`: Feature, telemetry, and communication comparisons
- `DRONE_SETUP_GUIDE.md`: Setup instructions for supported drones
- `docs/`: Research and supporting documentation

## Code of Conduct
Be respectful and constructive. See [Contributor Covenant](https://www.contributor-covenant.org/) for guidance.

## Getting Started
1. Review open issues and project documentation.
2. Comment on an issue to claim it or propose a new one.
3. Submit your changes via pull request.

---

## Issue Types
- **API Implementation:** Implement or refine the standard Java interface.
- **SDK Feature Mapping:** Map Tello-SDK and JCoDroneEdu features to the standard API.
- **Documentation:** Improve guides, comparisons, or research prompts.
- **Testing/Examples:** Add or update example code and test cases.

---

Thank you for helping make this project better!
